# 达妙履带接入说明

本文档描述第一阶段达妙履带电机如何作为 `can_driver` 的两轴 `velocity` 关节接入。

## 现状边界

- 协议后端：`DamiaoCan`
- 控制模式：仅 `velocity`
- 关节语义：仅履带/底盘轮关节
- 北向控制：通过 `diff_drive_controller`
- 上游速度入口：`/wheel_controller/cmd_vel`

第一阶段不在 `can_driver` 内保留 `/cmd_vel -> 左右履带速度` 的底盘业务逻辑，这部分已拆到 `mobility_control/base_cmd_node`。

旧 `control/src/damiao_diff_chassis_node.cpp` 已降级为归档参考实现，不再参与正式 bringup 或可执行构建。

## 关键文件

- `config/can_driver.yaml`
- `config/ros_controllers.yaml`
- `launch/can_driver.launch`

## 关节映射

当前默认使用两侧前轮关节作为正式硬件关节：

- `left_front_wheel_joint` -> 达妙左履带电机 `0x01`
- `right_front_wheel_joint` -> 达妙右履带电机 `0x02`

这样做的原因是：

1. 直接对齐现有 `car_urdf` 的 wheel joint 命名。
2. 便于后续把后轮作为 mimic/joint-state 跟随，而不是再引入一套非 URDF 关节名。

## 单独 bringup

```bash
roslaunch can_driver can_driver.launch use_controllers:=true
```

启动后可直接把标准底盘速度命令送给 `diff_drive_controller`：

```bash
rostopic pub -1 /wheel_controller/cmd_vel geometry_msgs/Twist \
  '{linear: {x: 0.2}, angular: {z: 0.0}}'
```

## 与正式执行链的关系

- `can_driver` 负责达妙协议后端、生命周期和 wheel joint 硬件接口
- `mobility_control` 负责 `/cmd_vel` 限幅与超时刹停
- `Eyou_ROS1_Master` 负责统一 lifecycle authority、正式 controller_manager 与 `diff_drive_controller`

正式链路推荐使用：

```bash
roslaunch Eyou_ROS1_Master hybrid_motor_hw.launch
```

## 参数来源

当前底盘几何参数沿用旧 `control` 包中的正式常量：

- `wheel_separation = 0.36`
- `wheel_radius = 0.075`
- `cmd_vel_timeout = 0.3`

如果后续实车标定结果更新，应同步修改：

- `config/ros_controllers.yaml`
- `Eyou_ROS1_Master/config/controllers_jtc.yaml`
